# DatePicker

A button-triggered calendar popover for picking a single date or a date
range. Styled with plain Tailwind utility classes resolved from a `size`
lookup table — same pattern as `Button`/`Select` — with heights, text sizes
and horizontal padding mirroring `Input`/`Select` exactly so it lines up
with other fields in a form grid. Supports outside-click dismissal and
month-by-month navigation. All date math uses plain JS `Date`, no date
library.

## Single date vs. range

Pass `value` + `onChange` for a single controlled date. Pass `range` +
`rangeValue` + `onChangeRange` for range-selection mode instead — never mix
the two sets of props.

```tsx
// Single date
const [value, setValue] = useState<Date | null>(null);
<DatePicker value={value} onChange={setValue} />

// Range
const [range, setRange] = useState<DateRange>({ start: null, end: null });
<DatePicker range rangeValue={range} onChangeRange={setRange} />
```

In range mode, the first click sets `start`, the second sets `end` (dates
are swapped automatically if the second click lands before `start`), and
the popover closes once both ends are picked. Hovering a day after `start`
is chosen previews the in-between range before `end` is committed.

## Composition

`DatePicker` forwards its ref to the trigger `<button>` element. Its root
DOM node is the wrapping `<div>` (label + trigger + calendar popover),
which is where `dataTestId` is applied as `data-test-id`. Use
`wrapperClassName` to style that outer wrapper and `className` to style the
trigger button itself — same split as `Select`.
