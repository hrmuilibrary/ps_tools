# TextArea

A labeled multi-line text input with helper/error messaging and an optional
character counter. Styled with plain Tailwind utility classes — no
variant-authoring abstraction on top, so consumer apps can override any part
of it via `className` (merged with `tailwind-merge`, last conflicting class
wins) rather than fighting a pre-hidden style layer.

## Error state

Pass `error` to mark the field invalid without a message, or `errorText` to
render an accessible error message (wired to the textarea via
`aria-describedby` and announced through `aria-invalid`). `errorText` also
implies the error state.

```tsx
<TextArea label="Description" error errorText="Description is required." />
```

## Character count

Pass `showCount` together with `maxLength` to render a live `n/max` counter
under the field.

```tsx
<TextArea label="Bio" showCount maxLength={200} />
```

## Overriding styles

Because `TextArea` resolves its `className` prop through the same `cn()`
merge utility the rest of the library uses, an app-level override always
wins over the defaults. Use `wrapperClassName` to style the outer
label/textarea/helper-text wrapper instead of the `<textarea>` element
itself.

```tsx
<TextArea className="rounded-full" wrapperClassName="max-w-md" />
```

## Composition

`TextArea` forwards its ref to the underlying `<textarea>` element and
spreads all native `<textarea>` props, so it composes with forms and any
other component in this library without a separate "as child" API.
