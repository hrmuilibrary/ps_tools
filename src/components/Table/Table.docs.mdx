# Table

A composable set of table primitives (`Table`, `THead`, `TBody`, `TR`, `TH`,
`TD`) plus a `DataTable` convenience wrapper that renders a fully-populated
table from a `columns`/`rows` config. Use the primitives directly for
hand-authored markup and full control over cell content; reach for
`DataTable` when the table is data-driven and column definitions are more
convenient than JSX.

## Composition

`Table` wraps its `<table>` in a horizontally-scrolling container so wide
tables don't blow out the layout. Because of that wrapper, `className` and
`ref` both target the `<table>` element itself (matching
`HTMLAttributes<HTMLTableElement>`), while `wrapperClassName` styles the
outer scroll container:

```tsx
<Table
  wrapperClassName="border-brand-500"
  className="[&_thead]:bg-brand-20"
>
  ...
</Table>
```

`density`, `striped`, `hoverable`, and `stickyHeader` are read via CSS
attribute selectors (`data-striped`, `data-hoverable`, `data-sticky`) set on
the `<table>` element itself, so `THead`/`TBody`/`TR` styling reacts to the
parent `Table`'s props without prop-drilling.

### Sortable columns

`TH` accepts `sortable`, `sortDir`, and `onSortChange`. It renders a sort
icon (ascending / descending / neutral) and cycles
`asc -> desc -> null -> asc` on click; the parent owns the actual sort
state and re-orders `rows`/`children` in response to `onSortChange`.

### DataTable

`DataTable` composes `Table`/`THead`/`TBody`/`TR`/`TH`/`TD` from a
`TableColumn<T>[]` + `rows: T[]` pair. Each column can provide either a
`render` (full JSX per row) or a simpler `accessor` (a value to render
as-is), and the whole table renders an empty-state row automatically when
`rows` is empty.
