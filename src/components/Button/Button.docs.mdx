# Button

Triggers an action or event. Styled with plain Tailwind utility classes
resolved from `variant`/`size` lookup tables — no variant-authoring
abstraction on top, so consumer apps can override any part of it via
`className` (merged with `tailwind-merge`, last conflicting class wins)
rather than fighting a pre-hidden style layer.

## Overriding styles

Because `Button` resolves its `className` prop through the same `cn()`
merge utility the rest of the library uses, an app-level override always
wins over the variant defaults:

```tsx
<Button className="bg-green-600 text-fg-inverse hover:bg-green-700 rounded-full">
  Approve
</Button>
```

## Composition

`Button` forwards its ref and spreads all native `<button>` props, so it
composes with forms, tooltips, and any other component in this library
without a separate "as child" API.
