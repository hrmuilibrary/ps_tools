# Tabs

Switches between panels of content. `Tabs` owns the active-value state via
context and shares it with `TabList`, `Tab`, and `TabPanel` — the
sub-components don't manage state themselves, they just read the current
value and call `setValue` on click.

## Composition

`Tabs` is a compound component: render a `<TabList>` of `<Tab>` triggers,
followed by one `<TabPanel>` per tab (matched by `value`). An inactive
`TabPanel` unmounts rather than hiding via CSS, so its content isn't kept
alive in the DOM while hidden.

```tsx
<Tabs defaultValue="profile">
  <TabList>
    <Tab value="profile">Profile</Tab>
    <Tab value="billing">Billing</Tab>
  </TabList>
  <TabPanel value="profile">...</TabPanel>
  <TabPanel value="billing">...</TabPanel>
</Tabs>
```

## Controlled vs. uncontrolled

Uncontrolled usage passes `defaultValue` (or nothing, in which case the
first `<Tab>`'s value is used) and `Tabs` tracks the active value
internally. Controlled usage passes both `value` and `onValueChange`, and
the consumer owns the state entirely — same shape as an
uncontrolled/controlled form input.

```tsx
const [value, setValue] = useState('profile');

<Tabs value={value} onValueChange={setValue}>
  <TabList>
    <Tab value="profile">Profile</Tab>
    <Tab value="billing">Billing</Tab>
  </TabList>
  <TabPanel value="profile">...</TabPanel>
  <TabPanel value="billing">...</TabPanel>
</Tabs>
```

## Variants

`variant` (on `Tabs`, inherited by `TabList` and `Tab` via context) switches
between `underline` (default — a brand-colored indicator under the active
tab), `pills` (a rounded track with a raised active pill), and `boxed` (a
bordered segmented control).

## Icons & badges

`Tab` accepts an optional `icon` node rendered before the label, and a
`badge` node (e.g. an unread count) rendered after it — hidden entirely
when `badge` is `undefined`, `null`, or `false`. Both recolor to match the
tab's active state.

## Disabled tabs

`Tab` accepts a native `disabled` attribute, which blocks selection and
applies the same `disabled:opacity-50 disabled:cursor-not-allowed`
treatment used elsewhere in the library.

## Overriding styles

`Tabs`, `TabList`, `Tab`, and `TabPanel` all resolve their `className` prop
through the same `cn()` merge utility the rest of the library uses, so an
app-level override always wins over the component defaults.

## Ref forwarding

`Tabs`, `TabList`, and `TabPanel` forward their ref to their own root
`<div>`; `Tab` forwards its ref to its root `<button>`. All four spread
their remaining native attributes, so they compose with the rest of the
library without a separate "as child" API. `TabList`, `Tab`, and `TabPanel`
must always be rendered inside a `Tabs` — they read the active value from
context and throw if that context is missing.
