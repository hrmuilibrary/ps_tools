# Switch

A toggle control for binary on/off settings, built on a native `<input
type="checkbox" role="switch">` styled with the `peer` variant so the
visible track and thumb react to the (visually hidden) input's checked,
disabled, and focus-visible state — no extra state management required.

## Label & description

Pass `label` and/or `description` to render them alongside the control,
wrapped together in a single `<label>` so clicking either the text or the
control toggles it. Omit both to render just the bare control (useful when
the surrounding layout supplies its own label, e.g. inside a settings row).
`labelPosition="left"` flips the control to the end of the row while
keeping the label left-aligned.

## Overriding styles

Because `Switch` resolves its `className` prop through the same `cn()`
merge utility the rest of the library uses, an app-level override always
wins over the defaults. `className` applies to the outer `<label>` wrapper
(spacing, alignment) rather than the track itself — style the track/thumb
by wrapping `Switch` instead of trying to override `bg-brand-500` etc.
inline.

## Composition

`Switch` forwards its ref to the underlying `<input>` element (not the
outer `<label>`), so consumers can call `.focus()` or read `.checked`
directly, matching how every other native form control in this library
behaves. It spreads all native `<input>` props (aside from `size` and
`type`, which the component owns), so it works uncontrolled
(`defaultChecked`) or controlled (`checked` + `onChange`) like any
checkbox.
