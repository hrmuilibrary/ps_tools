# Drawer

A panel that slides in from an edge of the viewport to show supplementary
content — forms, details, filters — without navigating away from the page.
It is a controlled component: the consumer owns `open` state and is notified
via `onClose` when the drawer wants to close (Escape key, overlay click, or
the built-in close button); it does not close itself.

Rendered through `createPortal` into `document.body`, so it always overlays
the rest of the page regardless of where it's mounted in the tree, and it
locks `document.body` scroll and moves focus onto the panel while open,
restoring the previously focused element on close.

## Behavior

- **Controlled open state** — `open` and `onClose` are required; the
  component renders nothing (`null`) when `open` is `false`.
- **Dismissal** — Escape key, clicking the overlay backdrop (unless
  `closeOnOverlayClick` is `false`), or the close button (hidden via
  `showClose={false}`) all call `onClose`.
- **Side & size** — `side` picks the edge (`right` default, `left`, `top`,
  `bottom`); `size` maps to a fixed width for left/right or a fixed height
  for top/bottom.
- **Focus & scroll** — body scroll is locked while open, and the panel
  receives focus on open, restoring focus to the previously focused element
  on close.

## Overriding styles

`className` is merged onto the panel via `cn()`, so an app-level override
always wins over the defaults:

```tsx
<Drawer open={open} onClose={() => setOpen(false)} className="bg-bg-container">
  ...
</Drawer>
```

## Composition

`Drawer` forwards its ref to the panel `<div>`, so it composes with anything
in this library that expects a DOM node ref (e.g. focus-trap or
click-outside utilities) without a separate "as child" API.
