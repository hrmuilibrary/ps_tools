# IconTile

A fixed-size container for a `lucide-react` icon, colored by a semantic
`tone`. Used to introduce list rows, empty states, and stat cards with a
consistent icon treatment instead of ad-hoc icon + background pairings.
Styled with plain Tailwind utility classes resolved from `tone`/`size`
lookup tables, same approach as `Button`.

## Accessibility

`IconTile` renders `aria-hidden="true"` by default, since it's typically
decorative alongside adjacent text (a list item title, a card heading,
etc.). Pass an `aria-label` when the tile is the only accessible
description of what it represents — doing so also removes `aria-hidden`:

```tsx
<IconTile icon={Bell} tone="brand" aria-label="Notifications" />
```

## Overriding styles

Because `IconTile` resolves its `className` prop through the same `cn()`
merge utility the rest of the library uses, an app-level override always
wins over the tone defaults:

```tsx
<IconTile
  icon={CheckCircle}
  tone="success"
  className="bg-green-100 dark:bg-green-900/30"
/>
```

## Composition

`IconTile` forwards its ref to the root `<div>` and spreads all native div
props, so it composes with tooltips, list rows, and any other layout in
this library without a separate "as child" API.
