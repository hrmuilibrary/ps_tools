# PersonCard

Compact person tile — avatar + name + role inside a rounded card. Used as
nodes in an org chart, but the shape (small chip with face + identity)
shows up anywhere people are listed: directory search, mention pickers,
team callouts. Centralising it avoids the same markup being rewritten in
each context.

## Variants

- **default** — neutral border, for ordinary rows in a list.
- **highlight** — brand-green tint + border, for "you" / the current user
  / a featured person.
- **warning** — amber border, for a person that needs attention.

<Stories />

## Clickable cards

Passing `onClick` renders the card as a `<button>` instead of a `<div>` so
it's focusable and clickable, while keeping the same visual layout.

```tsx
<PersonCard name="Ana Petrosyan" role="Product Designer" onClick={() => openProfile()} />
```

## Composition

`PersonCard` composes with `Avatar` (from `../Avatar`) for the face/initials
chip, and forwards its ref and any native element props to the root node
(`div`, or `button` when `onClick` is set), so it merges into lists, grids,
and org-chart layouts without a separate wrapper.
