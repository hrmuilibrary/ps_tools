# Status

Displays the current state of an entity (a record, a job, a request) as a
small pill, with a color dot by default. Styled with plain Tailwind utility
classes resolved from `variant`/`size` lookup tables — no variant-authoring
abstraction on top, so consumer apps can override any part of it via
`className` (merged with `tailwind-merge`, last conflicting class wins)
rather than fighting a pre-hidden style layer.

## The status dot

By default `Status` renders a small color dot before the label, matching
the variant's accent color. Passing `leftIcon` replaces the dot with the
icon; passing `dot={false}` (with no `leftIcon`) removes it entirely:

```tsx
<Status variant="success" dot={false}>
  Approved
</Status>
```

## Overriding styles

Because `Status` resolves its `className` prop through the same `cn()`
merge utility the rest of the library uses, an app-level override always
wins over the variant defaults:

```tsx
<Status dot={false} className="bg-purple-100 text-purple-900 rounded-md">
  Custom
</Status>
```

## Composition

`Status` forwards its ref to the root `<span>` and spreads all native
`<span>` props, so it composes with tooltips and other components in this
library without a separate "as child" API.
