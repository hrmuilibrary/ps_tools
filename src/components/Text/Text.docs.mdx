# Text

Renders any typographic string in the design system. `variant` maps onto the
semantic typography scale defined in `tokens.css` (`text-h-*`, `text-p-*`,
`text-label-*`) rather than Tailwind's default `xs/sm/base/lg` scale, and
`color` maps onto the semantic `text-fg-*` aliases in `theme.css` so it
adapts automatically under `.dark`.

## Polymorphism via `as`

Each `variant` has a sensible default element (e.g. `h-lg` renders an
`<h1>`, `p-std` renders a `<p>`, `label-md` renders a `<span>`), but any
element or component can be substituted with the `as` prop:

```tsx
<Text as="label" variant="label-std" htmlFor="email">
  Email address
</Text>
```

The ref always points at the actual rendered element, whatever `as`
resolves to.

## Overriding styles

Because `Text` resolves its `className` prop through the same `cn()` merge
utility the rest of the library uses, an app-level override always wins
over the variant defaults:

```tsx
<Text className="italic underline">Custom styled text</Text>
```

## Composition

`Text` forwards its ref and spreads all native attributes for whatever
element it renders as, so it composes with forms, tooltips, and any other
component in this library without a separate "as child" API.
